# Project: Asset Policy & Claim Management System

## Overview
Build a full-stack web application to manage asset policies and claims. The system should allow admins to upload policies with Excel asset data and allow users to create claims with auto-fetched asset details.

---

## Platform & Tech Stack
- Platform: Web Application
- Architecture: Full-stack
- Backend: Node.js (Express) OR Python (Flask)
- Frontend: React (preferred) or HTML/CSS/JS
- Database: PostgreSQL (preferred) or SQLite
- Hosting: Replit (Cloud)

---

## MODULE 1: Policy Management (Settings)

### Features:
- Create Policy with:
  - Policy Number (unique, required)
  - Start Date
  - End Date
  - Upload Policy PDF
  - Upload Asset Excel File
  - Active/Inactive Toggle

### Logic:
- Only ONE policy can be active at a time
- Activating a policy automatically deactivates others
- System always uses:
  - Active policy
  - Its Excel asset data

---

## MODULE 2: Excel Upload & Processing

### Required Excel Columns:
- Sr No.
- Inventory No.
- Asset No
- Capitalization Date
- Asset Class
- Asset Class (Short Name)
- Asset Description
- Asset Type
- Asset Sub Type
- SBU
- Plant
- Quantity
- Cost (Local)
- NBV (Local)
- Condition
- Asset Criticality
- Inventory On
- LCD Serial No
- IT Serial No
- Processor
- HDD
- RAM
- Model

### Requirements:
- Validate Excel before upload:
  - All required columns must exist
- Reject invalid files with clear error
- Handle duplicate records (Asset No / Inventory No)
- Store data linked to policy
- Use bulk insert for performance
